Test Plan  Zomato Food Delivery Application

Objective
The objective of this test plan is to verify that the Zomato food delivery web application works correctly for end users. The testing focuses on searching restaurants, ordering food, making payments, and tracking orders.

Scope
This test plan covers manual testing of user-facing functionalities of the Zomato web application. Backend testing and automation testing are out of scope.

Test Approach
Manual test cases are designed based on real user scenarios. Each test case is executed manually, and the actual result is compared with the expected result. Any deviation is logged as a defect using Jira.

Entry Criteria
- Application should be accessible
- Test data should be available
- Test cases should be prepared

Exit Criteria
- All planned test cases are executed
- Critical and high-severity defects are logged
- Test results are documented

Deliverables
- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- Jira Defect Logs

Roles and Responsibilities
- Tester: Test case execution and defect reporting
- Developer: Fixing reported defects
- QA Lead: Reviewing test results and closing defects

Tools Used
- Excel (Test cases and bug reports)
- Jira (Defect tracking)

Risks and Assumptions
- Payment testing is done using test or dummy payment details
- Application behavior may vary based on network conditions
